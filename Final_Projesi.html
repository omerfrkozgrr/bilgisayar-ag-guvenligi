// ANALİZ — NEDEN BAĞLANAMADK?

Port 23 Nmap tarafından tcpwrapped olarak işaretlenmesine rağmen doğrudan bağlantı reddedilmiştir. Bu durum cihaz üzerinde yerleşik bir ACL (Erişim Kontrol Listesi) bulunduğunu — muhtemelen yalnızca servis sağlayıcının yönetim IP'lerine izin verildiğini — göstermektedir. Ancak protokolün kendisi şifresiz (clear-text) iletişim kullandığından, portun ağda açık bulunması her halükarda kritik bir zafiyettir. ACL kaldırıldığı veya devre dışı bırakıldığı anda tüm trafik açık metin olarak dinlenebilir.

SAVUNMA
ANALİZ DROP Yeterli mi? — Loglama Gerekliliği

Mevcut kurallar paketleri sessizce DROP etmektedir. Bir saldırgan sürekli sisteme vursa bile sistem yöneticisinin bundan haberi olmaz. Bu ciddi bir görünürlük sorunudur.

❌ Mevcut Durum

iptables -A INPUT -p tcp --dport 23 -j DROP
# Sessiz düşürme — kimse görmez

✅ Önerilen Çözüm

iptables -A INPUT -p tcp --dport 23 -j LOG --log-prefix "TELNET_BLOCKED: "
iptables -A INPUT -p tcp --dport 23 -j DROP
# Önce logla, sonra düşür
// EK ÖNERİLER
fail2ban — Belirli bir IP'den çok fazla istek gelirse otomatik geçici ban uygular
rsyslog/syslog — iptables loglarını merkezi log sunucusuna iletir, kalıcı kayıt sağlar
SIEM entegrasyonu — Loglar analiz edilerek anormal trafik pattern'leri otomatik tespit edilir
NOT Dashboard & Topoloji Hakkında

Bu rapordaki ağ topolojisi ve iptables log akışı, gerçek Nmap ve iptables verilerinin görselleştirilmesidir. Canlı bir sistemde bu veriler bir backend API üzerinden beslenirdi — örneğin Python Flask + python-nmap kütüphanesi ile anlık ağ taraması yapılıp veriler frontend'e aktarılabilir. Bu projede tarayıcı güvenlik kısıtlamaları nedeniyle statik veri simülasyonu kullanılmıştır.
